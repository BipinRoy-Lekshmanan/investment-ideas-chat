# Deploying to Render

Render has a free tier, doesn't ask for a credit card, and can run this app
with no code changes. On the free tier, the app goes to sleep after 15
minutes without visitors; the next visitor wakes it up, which takes 30-60
seconds. After that it responds normally.

## What you'll need

- A GitHub account
- Your OpenAI API key
- The contents of `data/document.txt` (generated by running
  `python prepare_document.py your-report.pdf` locally first)

## Step 1: Push the code to a public GitHub repo

This repo is meant to be public — it's a portfolio piece. Because
`data/*.pdf` and `data/*.txt` are gitignored, pushing everything is safe;
the document itself won't be included.

```bash
git init
git add .
git commit -m "Initial commit"
```

Then create a new repo at [github.com/new](https://github.com/new) (public,
no README — you already have one), and:

```bash
git remote add origin https://github.com/<your-username>/investment-ideas-chat.git
git branch -M main
git push -u origin main
```

## Step 2: Create a Render account

1. Go to [render.com](https://render.com), click **Sign In**, and sign up
   with your GitHub account.
2. Verify your email if asked. No credit card required.

## Step 3: Create the web service

1. From the Render dashboard, click **New +** → **Web Service**.
2. Connect your GitHub account if prompted, and give Render access to your
   `investment-ideas-chat` repository.
3. Select the repository.
4. Fill in the settings:
   - **Language**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python app.py`
   - **Instance Type**: Free
5. Don't click Deploy yet — add the environment variables and secret file
   below first.

## Step 4: Environment variables

Under **Environment Variables**, add:

| Key | Value |
|---|---|
| `OPENAI_API_KEY` | your OpenAI key |
| `GRADIO_SERVER_NAME` | `0.0.0.0` |
| `GRADIO_SERVER_PORT` | `10000` |
| `DOCUMENT_TEXT_PATH` | `/etc/secrets/data/document.txt` |

Render expects your app to listen on port 10000. Gradio reads the first two
variables at startup to know where to bind — no code changes needed.

The last one matters because of how Render's Secret Files work — see Step 5.

Optionally also set `DOCUMENT_TITLE` if you want a name other than
"Investment Ideas Report".

## Step 5: Upload the document as a Secret File

This is the step that keeps the report itself off GitHub.

1. Still on the Environment tab, scroll to **Secret Files** and click
   **+ Add Secret File**.
2. **Filename**: `data/document.txt`
3. **Contents**: open your local `data/document.txt` and paste its full
   contents in.
4. Save. (Render's combined Secret Files limit is 1 MB — the extracted text
   from a ~100-page report is typically well under that. If yours is
   larger, see the "Scaling past full-context" section in
   [ARCHITECTURE.md](ARCHITECTURE.md) about switching to retrieval instead
   of shipping the whole document.)

Render doesn't write Secret Files to your repo's working directory — it
always mounts them under `/etc/secrets/`, preserving whatever path you typed
as the filename. So a Secret File named `data/document.txt` actually ends up
on disk at `/etc/secrets/data/document.txt`, not `data/document.txt`. That's
why Step 4 sets `DOCUMENT_TEXT_PATH` to that `/etc/secrets/...` path — it
tells `context.py` where to actually look. If you name the secret file
something else, update `DOCUMENT_TEXT_PATH` to match (same relative path,
`/etc/secrets/` prefix).

## Step 6: Deploy

1. Click **Deploy Web Service**.
2. Watch the logs — the first build takes a few minutes.
3. Once you see the Gradio startup message, your app is live at a URL like
   `https://investment-ideas-chat-xxxx.onrender.com`.
4. Open it and ask it something about your document.

Add that URL to the top of `README.md` and to your LinkedIn project entry.

## Updating the app

- **Code changes**: push to GitHub as usual — Render redeploys automatically.
- **Document changes**: regenerate `data/document.txt` locally, then paste
  the new contents into the same Secret File on Render and save (this
  triggers a redeploy on its own).

## Troubleshooting

- **The page takes ages to load.** Free-tier cold start after 15 minutes of
  inactivity — normal, resolves in under a minute.
- **Build fails.** Check the logs; usually a typo in the build/start command
  or a missing file.
- **App builds but errors on first message.** Almost always
  `OPENAI_API_KEY` missing or mistyped, or the Secret File wasn't saved with
  exactly the filename `data/document.txt`. Check the Environment tab, then
  use **Manual Deploy** to restart.
- **"FileNotFoundError: data/document.txt not found"** in the logs — almost
  always means `DOCUMENT_TEXT_PATH` isn't set to the `/etc/secrets/...` path
  described in Step 5. Double check the env var matches the Secret File's
  filename exactly, then trigger a Manual Deploy.
